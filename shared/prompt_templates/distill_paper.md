# 文獻蒸餾協定（Distill Paper Protocol）

> 適用場景：將單篇學術文獻拆解為可重複使用的「蒸餾卡」，供 CH2 文獻探討與縱橫分析使用。

## 輸入
1. 目標文獻的 PDF 或全文
2. `00_overall_review/` 中與本文獻相關的理論段落
3. `shared/glossary.md` → 確認術語對應

## 輸出格式（蒸餾卡）

```markdown
## 文獻：作者（年份）
- **研究問題**：
- **理論基礎**：
- **方法**：
- **核心發現**：
- **與本研究的關係**：
- **可引用的論點（直接摘句 + 頁碼）**：
- **批判或缺口**：
```

## CIMO 拆解選項（適用於實證文獻）
```
- C-Context（情境）：研究場域
- I-Intervention（干預）：策略行動
- M-Mechanism（機制）：因果邏輯
- O-Outcome（結果）：觀察到的效應
```

## 輸出規範
- 每張蒸餾卡 300-500 字
- 必須包含「與本研究的關係」欄位
- 引用句必須標註頁碼
- 批判欄位必須指出：這篇文獻「沒有回答什麼」或「在什麼情境下不適用」
- 檔案命名：`作者年份_關鍵詞.md`

## 禁止事項
- 只做摘要，不做評論（必須有批判欄位）
- 忽略「與本研究的關係」（這是篩選文獻的標準）
