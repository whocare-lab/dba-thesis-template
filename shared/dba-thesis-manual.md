## 總覽：CH1-CH5 與研究助理的角色

| 章節 | 研究助理能做什麼 | 需要你提供什麼 |
|------|----------------|--------------|
| CH1 緒論 | 協助撰寫研究背景、問題陳述、目的 | 你的實務觀察 + 初步研究主題 |
| CH2 文獻探討 | 檢索文獻、CIMO 蒸餾、縱橫分析、缺口識別 | 核心理論方向 + 對蒸餾卡的覆核 |
| CH3 研究方法 | 協助個案設計、資料收集計畫、品質檢核表 | 個案選擇邏輯 + 訪談對象清單 |
| CH4 個案分析 | CIMO 框架分析、命題歸納、跨個案比對 | 個案資料（訪談逐字稿、文件、檔案） |
| CH5 結論 | 理論貢獻歸納、實務建議、研究限制 | 對研究發現的反思 + 實務見解 |

> ⚠️ AI 是你的**研究助理**，不是你的**替代大腦**。你的實務經驗、產業洞察、與指導教授的對話，是核心資產。

## CH1：緒論

### 目的
建立研究的「正當性」：為什麼這個問題值得研究？為什麼現在？為什麼是你？

### 結構要素

```
1.1 研究背景與動機（Research Background）
    - 實務現象：你觀察到什麼？
    - 理論缺口：既有理論無法解釋什麼？
1.2 研究問題（Research Questions）
    - 主問題 + 2-3 個子問題
1.3 研究目的與預期貢獻
    - 理論貢獻（對管理學術）
    - 實務貢獻（對產業）
1.4 研究範圍與限制
1.5 論文結構
```

### AMJ 選題五大標準（CH1 自我檢核）

| 標準 | 核心問題 |
|:---|:---|
| **重要性** | 是否挑戰重大挑戰？ |
| **新穎性** | 是否改變對話方向？ |
| **奇特性** | 是否激發好奇心？ |
| **範疇廣度** | 視角是否夠宏大？ |
| **實用性** | 能否提供實務洞見？ |

## CH2：文獻探討

### 目的
建立研究的「理論基礎」：誰說過什麼？他們漏了什麼？你的研究補什麼？

### 三階段流程

#### 階段 A：文獻檢索（Literature Search）
- 工具：Crossref API + Semantic Scholar API（免費 HTTP）
- 過濾條件：被引 > 50、主流期刊、近 5 年
- 經典文獻豁免：奠基文獻不受年份限制
- 產出：文獻檢索清單（APA 引用 + DOI 連結 + 理論分類）

#### 階段 B：文獻蒸餾（Literature Distillation）
- 工具：CIMO 蒸餾法
- 輸入：階段 A 的 15-25 篇核心文獻
- 產出：蒸餾卡（Context / Intervention / Mechanism / Outcome）
- 每張卡你只需寫 50-100 字「與本研究的連結」

#### 階段 C：縱橫分析（Cross-Analysis）
- 橫向比較：同理論內文獻的 CIMO 一致性/矛盾
- 縱向演進：理論從奠基到最新的演變
- 跨理論對話：理論間互補或衝突
- 缺口矩陣：哪些 CIMO 組合尚未被研究

### 經典文獻豁免通道
- 被引 > 1000 → 自動豁免
- 理論原創書籍章節 → 人工審核
- 產業實務報告 → 標記 [Industry Source]

### 理論框架可視化：核心變數樹狀圖

建立理論框架時，建議使用樹狀結構將變數層級化：

```
                【研究目標】
          [你的論文題目]
                     │
       ┌─────────────┼─────────────┐
       │             │             │
   【外部衝擊】    【組織條件】    【策略行動】    【轉型結果】
       │             │             │             │
   [衝擊A]  [衝擊B]  [變數A]  [變數B]  [策略A]  [結果A]
```

**箭頭流向**：外部衝擊 → 組織條件 → 策略行動 → 轉型結果

> 用途：CH2 理論框架節、口試簡報第二張投影片、overall_review.md §2.2。

## CH3：研究方法

### 目的
說服讀者你的研究設計足夠嚴謹，足以回答 CH1 提出的研究問題。

### 結構要素

```
3.1 研究哲學與方法論立場
3.2 個案研究法（Case Study Method）
    3.2.1 個案選擇邏輯（Case Selection）
    3.2.2 個案背景描述
3.3 資料收集方法
    3.3.1 初級資料（訪談）
    3.3.2 次級資料（文件、檔案、年報）
3.4 資料分析方法
    3.4.1 CIMO 框架應用
    3.4.2 命題歸納邏輯
3.5 研究品質檢核
    3.5.1 構念效度（Construct Validity）
    3.5.2 內部效度（Internal Validity）
    3.5.3 外部效度（External Validity）
    3.5.4 信度（Reliability）
3.6 研究倫理
```

### 訪談設計
- 基於 CIMO 框架設計半結構式訪談大綱
- Context 相關問題、Intervention 相關問題、Mechanism 相關問題、Outcome 相關問題
- 訪談文件五件組：邀請信 → 同意書 → 記錄表 → 感謝信 → 逐字稿確認

### 關鍵方法論文獻

| 文獻 | 用途 |
|------|------|
| Yin (2018). Case Study Research (6th ed.) | 個案研究設計聖經 |
| Eisenhardt (1989). Building theories from case study research | 多個案理論建構 |
| Eisenhardt & Graebner (2007). Theory building from cases | 單個案也可以建構理論 |
| Langley (1999). Strategies for theorizing from process data | 過程資料分析 |
| Gioia et al. (2013). Seeking qualitative rigor | 質性嚴謹度 |

## CH4：個案分析

### 目的
呈現你的研究發現：用資料回答 CH1 的研究問題。

### 結構要素（單個案）

```
4.1 個案背景與脈絡
    4.1.1 公司沿革與組織架構
    4.1.2 產業環境與競爭態勢
    4.1.3 關鍵事件時間軸
4.2 個案分析（CIMO 框架）
    4.2.1 脈絡因素（Context）分析
    4.2.2 策略行動（Intervention）分析
    4.2.3 機制解釋（Mechanism）分析
    4.2.4 結果評估（Outcome）分析
4.3 命題歸納
4.4 與既有理論的對話
```

### CIMO 命題格式

> 在 [Context] 情境下，企業透過 [Intervention]，因為 [Mechanism] 的緣故，導致 [Outcome] 的結果。

## CH5：結論與建議

### 目的
回答「So What?」— 你的研究對學術界和實務界有什麼價值？

### 結構要素

```
5.1 研究發現摘要
5.2 理論貢獻
    5.2.1 對 [理論 A] 的延伸
    5.2.2 對 [理論 B] 的修正
    5.2.3 跨理論整合貢獻
5.3 實務意涵
    5.3.1 對企業經理人的建議
    5.3.2 對產業政策的建議
5.4 研究限制
5.5 未來研究建議
```

### 關鍵問題
你的結論「補了 CH2 的哪個缺口？」
