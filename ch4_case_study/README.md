# 第四章：個案分析

> 目的：呈現研究發現——用資料回答 CH1 的研究問題。

## 核心流程

1. **時間軸建立**：個案關鍵事件編年表 → `timeline_draft.md`
2. **訪談資料編碼**：使用 CIMO 框架編碼逐字稿
3. **CIMO 分析**：每個維度分別分析，對照文獻蒸餾卡
4. **命題歸納**：從 CIMO 分析中提取可命題化的規律
5. **理論對話**：對照 CH2 的缺口矩陣，說明你的發現補了哪個缺口

## 命題格式

> 在 [Context] 情境下，企業透過 [Intervention]，因為 [Mechanism] 的緣故，導致 [Outcome] 的結果。

## 訪談記錄表

見 `interviews/interview_template.md`（含 CIMO 編碼欄）

## 與 AI 的協作

告訴你的研究助理：「幫我建立 [個案公司] 的時間軸，關鍵事件如下：［清單］」
→ 你的研究助理會生成時間軸、標註轉折點、對照你的理論框架。
